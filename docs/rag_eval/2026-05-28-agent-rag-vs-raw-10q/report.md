# Agent + RAG vs Raw Model 对照实验报告（10题）

实验时间：2026-05-28 17:18-17:27（Asia/Shanghai）

## 实验目的

本轮实验对比当前完整产品链路（Agent + RAG）与同一底层模型的纯模型回答，观察两者在专业医美问题与生活化咨询问题中的差异。

重点看五件事：安全边界、风险识别、证据引用、生活化可读性、以及系统稳定性。

## 实验设置

| 组别 | 接口 | 能力范围 |
| --- | --- | --- |
| Agent + RAG | `/api/v1/agent/chat` | Agent 编排、RAG 检索、query rewrite、HyDE、rerank、risk_flags、citations |
| Raw Model | `/api/v1/chat/raw` | 同一底层模型，多轮 session 能力，但无 Agent、无 RAG、无 citations |

每个问题都使用全新的左右 session，避免上下文污染。

为避免上次 `Nd:YAG` 中冒号触发 BM25 `to_tsquery` 语法错误，本轮问题使用更自然的中文表达，避开高风险标点形式。最终 10/10 题均成功，没有出现 500。

| ID | 类型 | 问题 |

## 量化结果

| 指标 | Agent + RAG | Raw Model |
| --- | --- | --- |
| 成功回答 | 10/10 | 10/10 |
| 500/系统错误 | 0/10 | 0/10 |
| 每题 citations | 5/题 | 0/题 |
| 总 citations | 50 | 0 |
| 有 risk_flags 的题 | 8/10 | 无结构化字段 |
| blocked | 0/10 | 无结构化字段 |
| 平均耗时 | 约 43.6 秒 | 约 29.4 秒 |

## 总体结论

Agent + RAG 的核心优势很清楚：它更重视风险识别、会给出结构化 risk_flags，并且每题都有可追溯 citation。尤其在黄褐斑、泪沟、咬肌肉毒、鼻部填充这几类高风险或证据型问题上，Agent + RAG 的回答更符合医美咨询产品的安全定位。

Raw Model 的优势是回答更快、语言更顺、更像普通咨询。但它没有可验证引用，也没有结构化风险字段；在一些题里会给出非常具体的操作、参数或看似正式的参考文献，这些内容无法由系统证据链支撑。

## 分题摘要

### q01 玻尿酸填充后皮肤发白发紫疼痛

Agent + RAG 给出 5 条引用，并识别出血管栓塞、视力受累、组织坏死等 risk_flags。回答没有被 blocked，而是以风险识别和立即就医为主。

Raw Model 也能给出完整急症处理思路，但没有 citations，也没有结构化风险字段。

评价：Agent + RAG 更适合产品使用，因为风险提示更可追溯；但应继续避免给出过细、可执行的处置剂量。

### q02 深肤色黄褐斑与激光反黑风险

Agent + RAG 成功返回，引用集中在调Q激光和黄褐斑治疗风险，citation 贴题度高。它强调深肤色、复发性黄褐斑、反黑风险、基础治疗和谨慎选择光电治疗。

Raw Model 内容完整，但仍然是无来源可核验的医学建议。

评价：这是本轮 Agent + RAG 表现较好的一题，也验证了自然中文表达能绕开上次 BM25 语法错误。

### q03 痤疮凹陷性瘢痕方案选择

Agent + RAG 有 5 条 citations，但 citation 质量不理想：多数命中肉毒/眼睑下垂相关文献，仅 1 条点阵 CO2 相关文献贴题。尽管回答内容大体围绕点阵激光、黄金微针和 PRP，但证据链跑偏。

Raw Model 的表格结构和可读性强，但没有真实 citation 约束。

评价：这是 RAG 检索质量的明显问题，需要加强 intent/category filtering。痤疮瘢痕问题不应命中肉毒文献。

### q04 泪沟填充风险与不适合人群

Agent + RAG 表现很好：5 条 citations 全部来自眶周/泪沟填充解剖风险相关文献，risk_flags 覆盖血管栓塞、Tyndall 效应、慢性水肿、结节、解剖不适合等。

Raw Model 也能回答，但缺少可追溯证据。

评价：这是 Agent + RAG 的强项场景，适合用于 demo。

### q05 咬肌肉毒如何降低脸垮和咀嚼无力

Agent + RAG 引用集中在咬肌、肉毒、bruxism 和超声引导注射相关文献，贴题度高。risk_flags 包括咀嚼无力、面部下垂、不对称。

Raw Model 的建议也完整，但更像通用科普，无结构化安全输出。

评价：Agent + RAG 更可控、更适合产品回答。

### q06 28岁法令纹是否直接填玻尿酸

Agent + RAG 的回答方向正确：没有直接建议“打法令纹”，而是强调成因判断、中面部支撑、动态纹、填充风险。risk_flags 包括血管栓塞、过度填充、结节/迁移。

但 citations 部分混入肉毒眼睑下垂、咬肌等不太相关文献，只有玻尿酸/透明质酸酶相关 citation 较贴。

评价：回答策略好，citation 质量中等，需要优化检索过滤。

### q07 婚纱照前提亮和毛孔

Agent + RAG 给出实用建议，但 citation 仍有跑偏：命中了肉毒、半面痉挛、cellulite 等文献，只有 IPL 相关 citation 较贴题。

Raw Model 在这类生活化低风险场景里表达更自然，项目建议也更贴合用户时间窗口。

评价：Agent + RAG 的生活化表达和检索意图识别都需要加强。此题不适合作为当前 RAG 强项展示。

### q08 想让鼻子更挺但担心栓塞失明

Agent + RAG 表现较好：citations 包含鼻部填充、血管并发症、非手术隆鼻和严重玻尿酸并发症。risk_flags 覆盖失明、皮肤坏死、感染/结节、过度填充。

Raw Model 也会提醒风险，但证据不可追溯。

评价：Agent + RAG 对高风险注射部位有明显优势。

### q09 红血丝/泛红是否适合光子嫩肤

Agent + RAG 命中了 rosacea、post-acne erythema、IPL 等部分相关文献，但也混入一条肉毒咬肌 citation。risk_flags 有玫瑰痤疮活动期、屏障受损、PIH、短暂红斑等方向。

Raw Model 回答更生活化，能自然提醒先判断是否玫瑰痤疮。

评价：Agent + RAG 可用，但 citation 过滤还不够严格。

### q10 水光针感染和结节风险

Agent + RAG 给出机构资质、无菌操作、产品合规、感染/结节风险等有用建议，risk_flags 覆盖感染、结节、过敏、血管损伤。但 citations 明显跑偏，命中多核苷酸、肉毒和乔雅登，缺少真正水光针/注射感染相关证据。

Raw Model 的生活化风险清单更直接。

评价：回答方向可用，证据链不足。需要为水光/mesotherapy/skin booster 方向补充更贴的文献或结构化知识。

## 主要发现

1. Agent + RAG 在高风险医美问题上优势明显，尤其是泪沟、鼻部填充、咬肌肉毒和黄褐斑。
2. Raw Model 在生活化表达上仍然更顺，但不可追溯，容易产生“看起来像引用”的内容。
3. 当前 RAG 在部分问题上存在 citation 跑偏，尤其是痤疮瘢痕、婚纱照提亮、水光针。
4. Agent 每题固定返回 5 条 citation，但“数量稳定”不等于“质量稳定”。报告里应重点看 citation relevance。
5. 避开冒号类英文缩写后没有复现 500，但 BM25 tsquery escaping 仍是必须修的工程问题。

## 建议下一步

1. 修复 BM25 查询构造，避免特殊字符导致 `to_tsquery` 报错；优先考虑 token sanitization 或 `plainto_tsquery` / `websearch_to_tsquery`。
2. 给检索增加 category/intent routing，例如痤疮瘢痕强制优先 procedure/acne scar，水光针优先 skin booster/mesotherapy/injection safety。
3. citation 输出增加 relevance threshold，低相关 citation 不要为了凑满 5 条而展示。
4. 为生活化问题单独优化 prompt，让 Agent 保持安全和引用优势，同时减少生硬感。
5. 补充水光针、skin booster、mesotherapy、术后感染、结节管理方向的 RAG 文献。
